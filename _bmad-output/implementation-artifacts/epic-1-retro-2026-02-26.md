## Challenges

1. **LLM context blindness in production environments** — When debugging CI/CD failures (especially Story 1.6 Firebase Hosting deploy), the LLM lacked visibility into GitHub secrets, deployed state, and cloud console. It defaulted to fixing code when the root cause was environmental (missing `FIREBASE_SA_KEY_JSON` secret). **This was the single biggest time sink in the epic.**
2. **Story files became stale after fixes** — Implementation fixes were documented as annotations ("we tried X but Y worked") rather than rewriting the story to reflect the correct approach. This confuses future LLMs reading the stories.
3. **Files not committed to git** — Flagged as HIGH in 3/6 stories (1.1, 1.2, 1.3). Same pattern repeated because no explicit git checkpoint existed in the story workflow.
4. **Tooling assumptions broken** — `adk new` command didn't exist (1.1), `shadcn init` exited with error on Windows (1.2), Firebase deploy required undocumented secret (1.6)
5. **Error handling gaps recurring** — Firestore failures returning non-canonical responses (1.4), `submitTask()` missing try/catch (1.5). Pattern appeared in every story with real business logic.
6. **Docker not testable locally** — Docker was unavailable during Epic 1 development. Dockerfile correctness was fully deferred to CI/CD. (Now resolved — Docker installed locally.)

---

## Key Insights

1. **💡 Story files = single source of truth** — When a fix is discovered during implementation, update the story file to reflect the correct approach as if it was the original plan. Don't annotate with "we tried X but Y worked." Future LLMs reading the story get clean, unambiguous instructions.
2. **💡 LLMs need production context for CI/CD debugging** — When debugging deploy failures, provide the LLM with: GitHub Actions logs, secrets list (names only), deployed service state, and cloud console errors. Without this context, the LLM fixes what it can see (code) even when the root cause is environmental.
3. **💡 Code review is non-negotiable** — Every single story had HIGH severity issues caught by review. Never skip code review, even for "simple" scaffold stories.
4. **💡 Docker is now available** — Local container verification should be a standard pre-CI check for Epic 2+.
